# 对话模式说明

本文档详细介绍 LangGraph 多智能体助手支持的 6 种对话模式。

## 模式概览

| 模式 | 特点 | 适用场景 |
|------|------|----------|
| **智能对话** | 标准 AI 对话，支持工具调用 | 日常问答、任务执行 |
| **流式对话** | 实时流式输出响应 | 需要看到思考过程的对话 |
| **RAG 问答** | 基于知识库回答 | 查询私有文档内容 |
| **RAG 流式** | 知识库问答 + 流式输出 | 需要实时查看的知识库查询 |
| **真实 RAG 流式** | 逐字流式输出知识库结果 | 大文档查询，需要逐步阅读 |
| **Plan-Execute** | 任务分解执行，先规划后执行 | 复杂多步骤任务 |

---

## 1. 智能对话

### 特点

- **工具调用支持**：AI 可以自主调用 16 种内置工具
- **完整响应**：等待 AI 生成完整回答后一次性展示
- **支持人工审核**：高危工具会触发审核流程
- **状态持久化**：对话历史自动保存

### 工作原理

```
用户输入 → LLM 推理 → 判断是否需要工具
    ↓
需要工具 → 调用工具 → 获取结果 → 继续推理
    ↓
生成最终回答 → 返回完整响应
```

### 适用场景

- 日常问答和闲聊
- 需要调用工具的任务（计算、翻译、搜索等）
- 不需要实时看到生成过程的场景

### 使用示例

```
用户：计算 1234 * 5678
AI：[调用 calculator 工具]
    1234 × 5678 = 7,006,652
```

## 2. 流式对话

### 特点

- **实时输出**：AI 生成的内容实时展示
- **打字机效果**：像真人打字一样逐字显示
- **支持工具调用**：与智能对话相同的工具能力
- **更好的用户体验**：用户感知响应更快

### 工作原理

```
用户输入 → LLM 推理 → 流式生成 Token
    ↓
通过 SSE 实时推送 → 前端逐字渲染
```

### 适用场景

- 长文本生成（文章、报告）
- 需要看到 AI 思考过程
- 对响应速度敏感的场景

### 使用示例

```
用户：写一段关于人工智能的简介
AI：[实时逐字输出]
人
人工
人工智能
人工智能是
人工智能是指...
```

---

## 3. RAG 问答

### 特点

- **知识库检索**：基于上传的文档回答问题
- **来源引用**：回答附带文档来源和引用
- **完整响应**：一次性返回完整答案
- **支持多种文档格式**：TXT、MD、PDF

### 工作原理

```
用户提问 → 检索知识库 → 获取相关文档片段
    ↓
LLM 基于检索内容生成回答 → 附带来源引用
```

### 适用场景

- 查询公司内部文档
- 基于私有知识库问答
- 需要准确引用来源的场景

### 使用示例

```
用户：公司的年假政策是什么？
AI：根据《员工手册.pdf》，年假政策如下：
    1. 入职满1年：5天年假
    2. 入职满3年：10天年假
    
    [来源: 员工手册.pdf, 匹配度: 0.92]
```

### 前置条件

1. 上传文档到知识库
2. 点击"更新知识库"建立索引
3. 选择"RAG 问答"模式创建对话

---

## 4. RAG 流式

### 特点

- **知识库 + 流式**：结合 RAG 准确性和流式体验
- **实时生成**：基于检索结果实时生成回答
- **来源引用**：同样支持来源标注
- **更好的阅读体验**：长回答可以边生成边阅读

### 工作原理

```
用户提问 → 检索知识库 → 获取相关文档
    ↓
流式生成回答 → 实时展示 → 附带来源
```

### 适用场景

- 基于知识库的长文本回答
- 需要实时查看的知识库查询
- 文档总结、分析报告

### 与 RAG 问答的区别

| 特性 | RAG 问答 | RAG 流式 |
|------|----------|----------|
| 响应方式 | 完整返回 | 流式输出 |
| 等待时间 | 较长 | 即时开始 |
| 适用长度 | 短回答 | 长回答 |
| 用户体验 | 一次性阅读 | 边生成边读 |

---

## 5. 真实 RAG 流式

### 特点

- **逐字流式**：最细粒度的流式输出
- **真实感强**：最接近真人打字的效果
- **适合长文档**：大段内容可以逐步阅读
- **知识库专用**：专为 RAG 场景优化

### 工作原理

```
用户提问 → 检索知识库
    ↓
分块处理检索结果 → 逐块流式输出
    ↓
每块内容逐字渲染 → 真实打字效果
```

### 适用场景

- 大型文档查询
- 需要仔细阅读的长回答
- 演示和展示场景

### 三种 RAG 模式对比

| 特性 | RAG 问答 | RAG 流式 | 真实 RAG 流式 |
|------|----------|----------|---------------|
| 输出方式 | 完整 | 流式 | 逐字流式 |
| 速度 | 较慢 | 快 | 最慢 |
| 阅读体验 | 一次性 | 边生成边读 | 逐字阅读 |
| 适用场景 | 短回答 | 长回答 | 超长回答 |

---

## 6. Plan-Execute（任务规划执行）

### 特点

- **先规划后执行**：LLM 先将复杂任务分解为有序步骤
- **逐步执行**：按计划逐步调用工具执行
- **动态调整**：执行失败时自动调整计划
- **流式输出**：支持实时查看执行进度

### 工作原理

```
用户输入复杂任务 → Planner 分解为步骤
    ↓
Executor 逐步执行 → 每步调用工具
    ↓
Replanner 检查结果 → 必要时调整计划
    ↓
汇总结果 → 生成最终回答
```

### 适用场景

- 需要多步推理的复杂任务
- 需要调用多个工具的综合性任务
- 需要清晰执行计划的场景

### 使用示例

```
用户：帮我调研并总结2024年AI行业发展趋势
AI：[制定计划]
    步骤1: 搜索2024年AI行业发展趋势
    步骤2: 整理搜索结果
    步骤3: 总结发展趋势
    [逐步执行...]
    根据调研结果，2024年AI行业发展趋势如下：...
```

### 与智能对话的区别

| 特性 | 智能对话 (ReAct) | Plan-Execute |
|------|-------------------|-------------|
| 推理方式 | 单步推理循环 | 先规划后执行 |
| 适用场景 | 简单到中等任务 | 复杂多步骤任务 |
| 计划可见性 | 推理过程透明 | 步骤计划透明 |
| 动态调整 | 每轮重新决策 | Replanner 调整 |

## 模式选择建议

### 按场景选择

| 场景 | 推荐模式 |
|------|----------|
| 日常问答 | 智能对话 / 流式对话 |
| 工具调用（计算、翻译等） | 智能对话 |
| 查询私有文档 | RAG 问答 / RAG 流式 |
| 文档总结 | RAG 流式 |
| 长报告生成 | 流式对话 / RAG 流式 |
| 演示展示 | 真实 RAG 流式 |
| 复杂多步骤任务 | Plan-Execute |

### 按内容长度选择

| 内容长度 | 推荐模式 |
|----------|----------|
| 短回答（< 100字） | 智能对话 |
| 中等长度（100-500字） | 流式对话 / RAG 问答 |
| 长回答（500-2000字） | RAG 流式 |
| 超长回答（> 2000字） | 真实 RAG 流式 |

### 按响应速度选择

| 速度要求 | 推荐模式 |
|----------|----------|
| 最快响应 | 流式对话 |
| 平衡 | 智能对话 / RAG 流式 |
| 准确优先 | RAG 问答 |

## 如何切换模式

### 创建对话时选择

1. 点击左侧 "新建对话"
2. 在弹出的对话框中选择 "对话模式"
3. 从下拉菜单选择所需模式
4. 点击 "创建"

### 模式切换限制

- 对话创建后**不能**切换模式
- 需要切换模式时，请创建新对话
- 不同模式的对话可以在左侧列表中同时存在

## 常见问题

### Q: 为什么 RAG 模式没有使用知识库？

检查以下几点：
1. 是否已上传文档到知识库
2. 是否点击了"更新知识库"按钮
3. 文档是否已成功索引（查看知识库状态）
4. 问题是否与文档内容相关

### Q: 流式模式卡顿怎么办？

可能原因和解决方法：
1. **网络问题**：检查网络连接
2. **LLM 响应慢**：更换更快的模型或服务商
3. **前端卡顿**：刷新页面或重启浏览器

### Q: 工具调用在哪些模式下可用？

工具调用仅在以下模式可用：
- ✅ 智能对话
- ✅ 流式对话
- ✅ Plan-Execute

以下模式不支持工具调用：
- ❌ RAG 问答
- ❌ RAG 流式
- ❌ 真实 RAG 流式

### Q: 人工审核在哪些模式下生效？

人工审核在**所有模式**下都生效，只要触发高危工具就会进入审核流程。

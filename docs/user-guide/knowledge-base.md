# 知识库使用指南

本文档详细介绍如何使用知识库（RAG）功能，包括文档上传、索引管理和查询技巧。

## 什么是 RAG

**RAG**（Retrieval-Augmented Generation，检索增强生成）是一种将大语言模型与外部知识库结合的技术。

### 为什么需要 RAG

| 问题 | 传统 LLM | RAG 解决方案 |
|------|----------|--------------|
| 知识时效性 | 训练数据有截止日期 | 实时检索最新文档 |
| 私有知识 | 无法访问内部文档 | 基于私有知识库回答 |
| 幻觉问题 | 可能编造信息 | 基于真实文档生成 |
| 来源追溯 | 无法验证信息来源 | 提供文档引用 |

### 本项目 RAG 特点

- **双层检索**：向量检索（ChromaDB）+ 知识图谱（Neo4j）
- **增量更新**：支持新增文档而不重建整个索引
- **来源引用**：每个回答都标注信息来源
- **多格式支持**：TXT、MD、PDF

## 快速开始

### 1. 上传文档

1. 点击左侧 "知识库" 标签
2. 点击 "上传文档" 按钮
3. 选择文件（支持多选）
4. 点击 "确认上传"

### 2. 更新知识库

上传文档后，需要建立索引：

1. 点击 "更新知识库" 按钮
2. 选择更新模式：
   - **增量更新**：只处理新增和修改的文档（推荐）
   - **全量更新**：重建整个知识库索引
3. 等待处理完成

### 3. 开始查询

1. 创建 "RAG 问答" 或 "RAG 流式" 对话
2. 输入与文档相关的问题
3. 查看带来源引用的回答

## 文档管理

### 支持的格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| 文本文件 | .txt | 纯文本内容 |
| Markdown | .md | 支持 Markdown 格式 |
| PDF | .pdf | 便携式文档格式 |

### 文档上传限制

- **单文件大小**：最大 50MB
- **总文档数**：无限制（受磁盘空间限制）
- **字符编码**：推荐 UTF-8

### 文档组织建议

```
rag/docs/
├── 产品文档/
│   ├── 产品手册.pdf
│   ├── API文档.md
│   └── 更新日志.txt
├── 公司制度/
│   ├── 员工手册.pdf
│   ├── 考勤制度.md
│   └── 报销流程.txt
└── 技术文档/
    ├── 架构设计.md
    ├── 部署指南.md
    └── 开发规范.txt
```

## 索引管理

### 索引流程

```
文档上传
    ↓
文档加载（TXT/MD/PDF）
    ↓
文本分块（800字符/块，100字符重叠）
    ↓
向量化（智谱 embedding-2）
    ↓
存储到 ChromaDB
    ↓
索引完成
```

### 更新模式

#### 增量更新（推荐）

- **原理**：计算文档 MD5，只处理新增和修改的文档
- **优点**：速度快，保留已有索引
- **适用**：日常添加新文档

#### 全量更新

- **原理**：删除所有索引，重新处理所有文档
- **优点**：彻底清理，解决索引损坏问题
- **适用**：索引异常、批量替换文档

### 查看索引状态

在知识库页面可以看到：
- 文档总数
- 已索引文档数
- 索引状态（就绪/更新中/错误）
- 最后更新时间

## 查询技巧

### 提问方式

#### ✅ 好的提问

```
"公司的年假政策是什么？"
"产品的主要功能有哪些？"
"如何部署这个系统？"
"根据文档，API 的认证方式是什么？"
```

#### ❌ 不合适的提问

```
"你好"  （与知识库无关）
"今天天气怎么样？"  （知识库外信息）
"总结一下"  （过于笼统，没有指定范围）
```

### 提高查询质量

1. **具体明确**：指明文档类型或范围
   ```
   "根据员工手册，年假有多少天？"
   ```

2. **关键词准确**：使用文档中的专业术语
   ```
   "GraphRAG 和 LightRAG 的区别是什么？"
   ```

3. **分步提问**：复杂问题拆分成多个问题
   ```
   Q1: "系统的硬件要求是什么？"
   Q2: "系统的软件依赖有哪些？"
   ```

### 理解回答

RAG 回答通常包含：

```
[回答内容]

[来源引用]
- 文档名：员工手册.pdf
- 匹配度：0.92
- 内容片段：...
```

**匹配度说明**：
- 0.9+：高度相关
- 0.7-0.9：相关
- 0.5-0.7：可能相关
- < 0.5：不太相关（可能是幻觉）

## 高级功能

### 知识图谱（可选）

如果配置了 Neo4j，可以启用知识图谱功能：

1. **实体关系抽取**：自动提取文档中的实体和关系
2. **图谱问答**：基于图结构进行推理
3. **可视化**：在 Neo4j Browser 中查看知识图谱

启用方法：
```bash
# .env
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PWD=your-password
ENABLE_KNOWLEDGE_GRAPH=true
```

### LightRAG 双层检索

系统支持三种检索模式：

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **local** | 仅向量检索 | 精确匹配查询 |
| **global** | 仅知识图谱 | 关系推理查询 |
| **hybrid** | 混合检索（默认） | 综合查询 |

切换模式：
```
用户：使用 global 模式查询
AI：[切换到知识图谱检索]
```

## 性能优化

### 索引优化

1. **合理分块**：保持文档段落完整
2. **清理无用内容**：删除页眉页脚、重复内容
3. **定期重建**：每月执行一次全量更新

### 查询优化

1. **使用增量更新**：避免全量重建
2. **控制文档数量**：单个知识库建议 < 1000 个文档
3. **及时清理**：删除不再需要的文档

## 故障排查

### 问题：文档上传失败

**可能原因**：
- 文件格式不支持
- 文件过大（> 50MB）
- 文件名包含特殊字符

**解决方法**：
1. 检查文件扩展名
2. 压缩或分割大文件
3. 重命名文件（使用英文、数字、下划线）

### 问题：索引更新失败

**可能原因**：
- 智谱 AI API Key 无效
- 网络连接问题
- 文档内容为空或损坏

**解决方法**：
1. 检查 `ZHIPUAI_API_KEY` 配置
2. 检查网络连接
3. 删除问题文档重新上传

### 问题：RAG 回答不准确

**可能原因**：
- 文档未成功索引
- 问题与文档不相关
- 检索阈值设置不当

**解决方法**：
1. 检查知识库状态，重新更新索引
2. 调整提问方式，使用文档中的关键词
3. 检查文档内容是否包含答案

### 问题：知识图谱无法使用

**可能原因**：
- Neo4j 未配置或连接失败
- `ENABLE_KNOWLEDGE_GRAPH` 设置为 false
- 未构建知识图谱

**解决方法**：
1. 检查 Neo4j 配置
2. 启用知识图谱功能
3. 点击 "构建知识图谱" 按钮

## 最佳实践

### 文档准备

1. **格式统一**：尽量使用 Markdown 格式
2. **结构清晰**：使用标题层级组织内容
3. **内容完整**：避免断章取义的片段
4. **定期更新**：及时更新过期文档

### 索引策略

1. **首次使用**：执行全量更新
2. **日常维护**：使用增量更新
3. **定期重建**：每月一次全量更新
4. **监控状态**：关注索引状态和错误日志

### 查询策略

1. **先测试**：用已知答案的问题测试知识库
2. **逐步优化**：根据回答质量调整文档和提问
3. **结合模式**：普通对话 + RAG 对话结合使用

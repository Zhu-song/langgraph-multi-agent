# 人工审核功能

本文档详细介绍人工审核（Human-in-the-Loop）机制，包括审核流程、配置方法和最佳实践。

## 什么是人工审核

人工审核是一种安全机制，当 AI 需要执行**高危操作**时，会暂停工作流并等待人工批准后才继续执行。

### 为什么需要人工审核

| 风险 | 说明 | 审核保护 |
|------|------|----------|
| **数据安全** | 防止敏感数据被意外修改或删除 | 文件删除、数据库写入需审核 |
| **网络安全** | 防止不当的网络访问 | 联网搜索需审核 |
| **成本控制** | 防止意外的 API 调用费用 | 高成本操作需审核 |
| **合规要求** | 满足审计和合规要求 | 所有高危操作留痕 |

## 高危工具列表

以下工具被定义为高危工具，调用前需要人工审核：

| 工具 | 风险等级 | 风险说明 |
|------|----------|----------|
| **web_search** | 🔴 高 | 联网搜索可能获取不当信息 |
| **file_delete** | 🔴 高 | 文件删除可能造成数据丢失 |
| **database_write** | 🔴 高 | 数据库写入可能破坏数据 |
| **file_write** | 🟡 中 | 文件写入可能覆盖重要内容 |
| **incremental_rag_operate** | 🟡 中 | 知识库更新影响查询结果 |

### 安全工具（无需审核）

以下工具为只读或低风险操作，无需审核：

- `calculator` - 数学计算
- `translate` - 翻译
- `summary` - 摘要
- `rag_knowledge_query` - 知识库查询（只读）
- `time_tool` - 时间查询
- `random_tool` - 随机数生成

## 审核流程

### 流程图

```
用户提问
    ↓
LLM 判断需要调用高危工具
    ↓
触发 interrupt() - 工作流暂停
    ↓
用户收到审核提示
    ↓
┌─────────────────┐
│   审核面板      │
│  ┌───┐ ┌───┐   │
│  │通过│ │拒绝│  │
│  └───┘ └───┘   │
└─────────────────┘
    ↓
通过 → 执行工具 → 返回结果 → 继续对话
拒绝 → 返回拒绝信息 → 继续对话
```

### 详细步骤

#### 1. 触发审核

当用户请求涉及高危操作时：

```
用户：搜索最新的人工智能新闻
AI：检测到高危工具调用 [web_search]
    正在等待人工审核...
```

#### 2. 处理审核

1. 点击左侧 "审核" 标签
2. 查看待审核请求详情：
   - 工具名称
   - 调用参数
   - 风险说明
   - 请求时间

3. 选择操作：
   - **通过**：允许执行工具调用
   - **拒绝**：拒绝执行并返回提示

#### 3. 查看结果

**通过情况**：
```
[审核已通过]
AI：[执行 web_search]
    根据搜索结果，最新的人工智能新闻包括：...
```

**拒绝情况**：
```
[审核已拒绝]
AI：抱歉，该操作已被拒绝。如需搜索功能，请联系管理员。
```

## 配置方法

### 启用/禁用审核

在 `.env` 文件中配置：

```env
# 启用人工审核（默认）
ENABLE_APPROVAL=true

# 禁用人工审核（不推荐用于生产环境）
ENABLE_APPROVAL=false
```

### 自定义高危工具

编辑 `config.py` 文件：

```python
# 高危工具列表
HIGH_RISK_TOOLS = {
    "web_search",
    "file_delete",
    "database_write",
    "file_write",
    # 添加自定义高危工具
    "your_custom_tool",
}
```

### 配置审核超时

```python
# workflow.py
APPROVAL_TIMEOUT = 3600  # 审核超时时间（秒）
```

## 最佳实践

### 安全建议

1. **生产环境必须启用审核**
   ```env
   ENABLE_APPROVAL=true
   ```

2. **限制审核权限**
   - 只有管理员可以处理审核
   - 普通用户只能查看自己的审核请求

3. **定期审计**
   - 查看审核日志
   - 分析通过/拒绝比例
   - 发现异常操作模式

### 用户体验优化

1. **提前告知用户**
   ```
   用户：删除文件 xxx.txt
   AI：该操作需要管理员审核，请稍候...
   ```

2. **快速响应**
   - 管理员及时处理审核请求
   - 设置审核超时提醒

3. **清晰说明**
   - 告知用户为什么需要审核
   - 说明审核预计时间

### 常见场景处理

#### 场景 1：误触发审核

**问题**：正常查询触发了 web_search 审核

**解决**：
1. 检查用户意图
2. 如果确实需要搜索，予以通过
3. 如果不需要，拒绝并建议修改提问

#### 场景 2：批量操作

**问题**：用户需要批量删除文件

**解决**：
1. 评估批量操作的风险
2. 要求用户确认文件列表
3. 逐条审核或批量审核

#### 场景 3：紧急情况

**问题**：紧急情况下需要快速执行高危操作

**解决**：
1. 建立紧急审核通道
2. 设置白名单用户
3. 临时禁用审核（仅限紧急情况）

## 故障排查

### 问题：审核面板不显示请求

**可能原因**：
1. `ENABLE_APPROVAL` 设置为 false
2. 没有触发高危工具
3. 审核请求已过期

**解决方法**：
1. 检查 `.env` 配置
2. 确认触发了高危工具
3. 检查审核超时设置

### 问题：审核通过后无响应

**可能原因**：
1. 工具执行失败
2. 网络连接问题
3. 工作流状态异常

**解决方法**：
1. 查看后端日志
2. 检查工具配置
3. 重启服务

### 问题：审核状态不同步

**可能原因**：
1. 多用户同时处理同一请求
2. WebSocket 连接断开
3. 前端缓存问题

**解决方法**：
1. 刷新页面
2. 重新登录
3. 检查网络连接

## API 接口

### 检查审核状态

```http
GET /api/approval/check/{user_id}
```

响应：
```json
{
  "has_pending": true,
  "pending_count": 2,
  "requests": [
    {
      "tool_name": "web_search",
      "args": {"query": "AI新闻"},
      "timestamp": "2024-01-15T10:30:00Z"
    }
  ]
}
```

### 处理审核

```http
POST /api/approval/resume
Content-Type: application/json

{
  "user_id": "user123",
  "decision": "yes",  // "yes" 或 "no"
  "tool_call_id": "call_xxx"
}
```
